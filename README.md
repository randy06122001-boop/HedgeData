# Perplex Alpha Database Release

This repository contains the `perplex.db` SQLite database, which is a snapshot of the historical market data, trade records, and multi-agent outputs generated by the Perplex Alpha platform.

> [!NOTE]
> The proprietary strategy logic, machine learning model weights, and backend configurations have been intentionally excluded from this repository to protect the underlying trading architecture ("the secret sauce").

## Database Schema Overview

The database uses SQLite and contains several interconnected tables tracking everything from raw market OHLCV data to LLM agent reasoning.

### `ohlcv`
Stores historical market data for the assets traded by the system.
- `symbol`, `timeframe`, `timestamp`
- `open`, `high`, `low`, `close`, `volume`
- `vwap`, `trade_count`

### `trades` & `orders`
Execution logs of the portfolio manager agent.
- `orders`: Tracks lifecycle of broker orders (limit/stop prices, fill status).
- `trades`: Confirmed executions including commissions, total value, and the specific `signal_id` that triggered the trade.

### `positions`
Currently held or closed positions, updating in real-time as market conditions change. Tracks `unrealized_pnl` and `realized_pnl`.

### `signals`
The output of the decision-making pipeline.
- `symbol`, `direction`, `strength`, `confidence`
- `source_agent`: The specific agent that generated the signal.
- `reasoning`: The textual explanation provided by the LLM for the trading signal.

### `agent_outputs`
Raw logs of LLM interactions across the multi-agent system (Sentiment Analyst, Fundamentals Analyst, Refiner, etc.).
- `agent_name`, `model_name`
- `prompt_tokens`, `completion_tokens`, `latency_ms`
- `output_text` and `structured_output`

### `portfolio_snapshots`
Time-series data of the entire portfolio's health, including total value, cash vs. invested ratio, and portfolio heat.

### `audit_log`
System-level event tracking for debugging and risk management (vetoes, errors, pipeline latency).

## Getting Started

You can query this database using any standard SQLite client, Python's `sqlite3` module, or pandas:

```python
import sqlite3
import pandas as pd

# Connect to the database
conn = sqlite3.connect('perplex.db')

# Load the trades into a DataFrame
df_trades = pd.read_sql_query("SELECT * FROM trades", conn)
print(df_trades.head())
```
